# Contributing

Thanks for your interest in contributing to SCQCS.

## Ways to Contribute

### Report Issues
- Website bugs or broken links
- Security concerns (see [security.txt](/.well-known/security.txt))
- Documentation errors or unclear language

### Suggest Improvements
- Additions to the security checklist
- Better patterns or practices
- Accessibility improvements

### Use as a Template
The best contribution is using these patterns in your own projects. This repository is designed to be forked and adapted. No permission needed.

## Guidelines

### For Website Changes
- Test locally before submitting
- Verify all links work
- Run through the [security checklist](SITE_SECURITY_CHECKLIST.md)

### For Checklist Updates
- Keep it practical and actionable
- Include specific commands or code snippets
- Cite sources for security recommendations

### For Framework Documentation
- Maintain the "not a product" stance
- Avoid absolute claims (see Legal Copy Guidelines in checklist)
- Keep language accessible to non-experts

## Not Accepted
- Marketing or promotional content
- Unverified security claims
- Changes that add tracking or analytics
